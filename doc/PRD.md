## 一、愿景与核心问题

### 1.1 要解决的根本问题

会议记录是知识工作的刚需，但现有工具有三个根本缺陷：

| 痛点 | 现状 | 根因 |
|------|------|------|
| 隐私风险 | 腾讯会议等工具云端转写，数据留厂商 | 商业利益驱动数据留存 |
| 设备限制 | 系统内录能力弱，macOS/Linux 支持差 | 跨平台实现成本高 |
| 工具割裂 | 会议→转写→编辑→分发，需要 4+ 个工具 | 没有一件事被真正解决 |

### 1.2 第一性问题

> 会议记录的本质是什么？

是**让没参会的人获得同等的上下文**。不是为了 AI，是为了**可查询、可追溯、可复用**。

所以产品目标不是「语音转文字」，而是：**一个会议，生成一份可操作的资产**。

### 1.3 核心假设

- 假设 1：moonshine-small 的准确率在安静内录场景下达 92%+（需实测）
- 假设 2：双路分别识别可以将「自己说」和「对方说」有效区分
- 假设 3：用户愿意为「隐私+离线」接受轻度配置成本

## 二、用户与场景

### 2.1 目标用户

| 用户类型 | 场景 | 核心诉求 |
|----------|------|----------|
| 远程工程师 | 参加跨国会议，技术讨论密集 | 低延迟、可导出、隐私 |
| 产品/运营 | 参加每日站会、周会 | 快速出文字稿、不打断讨论 |
| 研究人员 | 听播客/课程/讲座 | 实时字幕、内容结构化 |

### 2.2 使用场景

```
场景 A：腾讯会议 / Zoom 内录
  → 打开 App → 选内录设备 → 开始会议 → 会议结束 → 导出 MD

场景 B：Mac 上听播客
  → 选 BlackHole 内录 → 开始录制 → 实时字幕 → 复制重点

场景 C：双人访谈
  → 双路麦克风 → 分别转写 → 合并输出带时间戳的对话记录
```

## 三、MVP 功能定义

### 3.1 必须有（MVP）

| 功能 | 描述 | 验收标准 |
|------|------|----------|
| 麦克风转写 | 采集系统麦克风，实时流式转写 | 延迟 < 500ms |
| 系统内录 | WASAPI Loopback（Win）/ BlackHole（Mac） | 能录到扬声器声音 |
| 实时字幕 UI | 实时显示转写结果，区分来源 | 肉眼无感知延迟 |
| 导出 Markdown | 保存为 `.md` 文件 | 包含时间戳和来源标记 |
| 音频设备选择 | UI 选择输入设备 | 支持查看设备列表 |

### 3.2 应该有（v1.0）

| 功能 | 描述 | 优先级 |
|------|------|--------|
| VAD 人声检测 | 静音自动跳过，减少噪音 | P0 |
| 双路同时采集 | 麦克风 + 内录并行 | P1 |
| SRT 字幕导出 | 兼容视频播放器字幕格式 | P2 |
| 实时去重 | 去除连续重复输出 | P2 |

### 3.3 可以有（v1.x）

| 功能 | 描述 | 备注 |
|------|------|------|
| 说话人分离 | 双路情况下区分发言人 | 依赖双路物理隔离 |
| 标点恢复 | 加标点提高可读性 | 可接 whisper-punctuate |
| 热词自定义 | 提高垂直领域词汇准确率 | 配置文件 |
| WebDAV 同步 | 推送笔记到 Notion/飞书等 | 插件化 |
| WebUI | 支持手机扫码查看实时字幕 | Electron 网络功能 |

## 四、技术方案

### 4.1 架构决策

```
Electron 主进程
  ├─ UI 层（renderer/）         → HTML/CSS/JS，无框架依赖
  ├─ IPC 通道（preload.js）      → contextBridge 安全桥接
  └─ Python 子进程（transcribe.py）
       ├─ sounddevice 音频采集
       ├─ moonshine-small 转写
       └─ stdout JSON 回报
```

**决策：为什么不直接用 Node.js 做音频？**
- sounddevice 是 Python 生态，WASAPI/ CoreAudio / ALSA 适配成熟
- moonshine-small 是 Python 包
- Node.js 音频生态碎片化，维护成本高
- Python 子进程隔离好，崩溃不拖垮 UI

### 4.2 音频分块策略

```
采样率：16000 Hz（moonshine 要求）
分块：0.3 秒 / chunk = 4800 samples
推送频率：~3.3 FPS
RTF（实时因子）：< 0.5（CPU 推理 < 150ms）
```

### 4.3 数据流

```
[麦克风 16kHz] ──┐
                 ├──→ VAD 过滤 ──→ moonshine-small ──→ 去重 ──→ JSON stdout
[内录 16kHz] ──┘                                         │
                                                          ↓
                                              [Electron IPC] → [UI 实时渲染]
                                                          │
                                                          ↓
                                              [文件写入] → MD / SRT
```

### 4.4 依赖版本

| 依赖 | 版本 | 备注 |
|------|------|------|
| moonshine | ≥ 0.3.0 | 模型：moonshine-small |
| sounddevice | ≥ 0.4.6 | 跨平台音频 |
| numpy | ≥ 1.24.0 | 数值计算 |
| Electron | ≥ 28.0 | 主框架 |
| electron-builder | ≥ 24.9 | 打包 |
